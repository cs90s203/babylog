# 寶寶日誌 — 版本記錄

版號規則跟 JPL（日語學習 App）主 dev 對話一致：`MAJOR.MINOR.PATCH`——新功能升 MINOR、
架構級的重大改動升 MAJOR、bug 修正/小改善升 PATCH。版本號顯示在設定頁最下方。

## v2.2.1 — 2026-07-01

**Bug 修正：兩支手機登入後看不到同一份寶寶資料**

- 根因：裝置登入 Google 帳號後，即時監聽器只會把「遠端 Firestore 已有的資料」拉回本機
  （remote → local），但裝置在登入之前／登入當下就已經存在本機的資料，從來沒有被推上去過
  （local → remote 只有「登入後新增的紀錄」才會觸發 `Store._cloudPush`）。結果兩支手機各自
  的舊資料卡在自己身上，永遠不會同步給對方，看起來就像「沒連在同一個寶寶上」。
- 修法：登入成功那一刻，額外執行一次 `Sync._pushAllLocal()`，把當下本機所有 events/
  growth/settings 整批推上 Firestore（`merge:true`，重複執行也安全），確保不管這支手機
  是不是第一次登入、本機有沒有登入前就存在的資料，都會補齊上去。

## v2.2.0 — 2026-07-01

**改善：時間軸改成固定 24+3 小時捲動視窗，拖曳改成不觸發整頁重繪**

- 活動時間軸「今天」不再是「日曆上的今天（00:00–24:00）」，改成固定的「現在往前 24 小時
  ～ 往後 3 小時」捲動視窗。舊邏輯用「小時的第幾小時（0–24 的小數）」當座標，天生沒辦法
  跨午夜——過午夜看不到前一天的紀錄，拖曳也拉不過午夜線。新座標改用「距離視窗起點幾小時」
  （純小數，可以超過 24），實際換算靠真正的 `Date` 物件運算，跨天自然正確，不需要特別處理
  午夜。空檔壓縮功能（≥3 小時無紀錄自動收合）邏輯完全不變，只是套用在新座標上。
- 拖曳事件調整時間會抖動、往左拖會消失、拖曳一開始頁面會跳回頂端——三個症狀追到同一個根因：
  舊版拖曳時，手指每移動一像素就呼叫一次 `Store.updateEvent()`，觸發整個 App 重新渲染一次
  （`#root.innerHTML` 整個換掉）。畫面重繪那麼高頻，除了卡頓，捲動區域的 DOM 節點也被整個
  換成新的（重置捲動位置），拖曳用來換算座標的參考節點也可能抓到舊的/已卸載的節點，算出無效
  時間讓事件消失。修法：拖曳中只直接操作那顆事件標籤自己的 DOM 位置/文字（不寫入 Store、不
  觸發重繪），放開手指才真正寫入一次資料。額外加上：`render()` 現在會在整頁重繪前後保存/還原
  捲動位置，就算之後別的地方又不小心觸發重繪，也不會再無故跳回頂端。

## v2.1.0 — 2026-07-01

**新功能：下拉重新整理（純重整頁面，跟同步無關）**

- Firebase 版拿掉了舊版下拉同步手勢後，重新加回一個下拉重新整理功能——但這次純粹是重新載入
  整個頁面（`location.reload()`），不再嘗試觸發任何同步邏輯（Firestore 本來就即時同步，
  不需要靠這個手勢），單純給需要「重新開始乾淨狀態」時用（開發測試、或裝置的監聽器卡住時）
- 下拉時會有指示條浮現（「↓ 下拉重新整理」→ 拉超過門檻變「↓ 放開重新整理」），放開後短暫
  顯示「重新整理中…」再重新載入
- 指示條元素刻意掛在 `#root` 外面（直接接在 `body` 下），因為 `#root` 每次畫面重繪都會被
  整個換掉，如果指示條掛在裡面，手勢做到一半重繪就會把它砍掉重建

## v2.0.2 — 2026-07-01

**Bug 修正：iOS Safari 日期輸入框溢出卡片邊界**

- `<input type="date">` 在 iOS Safari 上會忽略 CSS `width:100%`，用自己內部的偏好寬度
  渲染，導致設定頁生日框、CSV 匯出「從/到」日期框都超出卡片邊界（Chrome 桌機版重現不出來，
  靠使用者提供的實機截圖才確認根因）
- 加上 `-webkit-appearance: none` 把版面控制權還給我們自己的 CSS，原生日期選擇器點擊後
  仍正常彈出

## v2.0.1 — 2026-07-01

**Bug 修正 + 改善：行動裝置手勢衝突、＋按鈕配色對齊規範**

- 長按活動時間軸的事件標籤會被 iOS 原生的選字/放大鏡搶走手勢，導致長按拖曳失效
  （`-webkit-touch-callout: none` + `user-select: none`）
- 下拉重新整理仍會誤觸發 Safari 原生的整頁重新整理：真正在捲動的元素是 `.ns`，不是
  `body`，`overscroll-behavior: contain` 要加在 `.ns` 上才擋得住
- 中間 ＋（記錄成長）按鈕改用設計規範指定的日/夜間專屬配色（`#b78d52` /
  `#9d8c6c`），不再沿用主色金黃

## v2.0.0 — 2026-07-01

**新功能：Firebase Google 登入 + Firestore 即時同步，取代 GitHub Token 方案**

- 拿掉「每支手機手動貼 GitHub token」的同步方式，改成 Google 帳號登入（Firestore
  Security Rules 白名單限制哪些帳號能存取）
- events / growth / settings 改成 Firestore 逐筆文件即時監聽（`onSnapshot`），不再需要
  下拉重新整理——換裝置幾秒內自動同步
- 拿掉舊的 pull-merge-push-retry 整包 JSON 合併邏輯，改成單筆文件 `updatedAt` 比較合併
- 設定頁「同步與帳號」改成登入/登出卡片
- 新增「下載備份 JSON」手動備份按鈕（Firestore 沒有 GitHub commit 那樣的版本歷史）

## v1.1.1 — 2026-07-01

**Bug 修正：時間軸拖曳手勢誤觸發整頁重繪，吃掉所有按鈕點擊**

- `main.js` 的全域 `pointerup` 監聽器不管有沒有真的在拖曳都會呼叫 `dragEnd()`，而
  `dragEnd()` 無條件觸發整頁重繪——重繪發生在瀏覽器「放開→觸發 click」中間，導致使用者
  點的按鈕在 click 真正觸發前就被砍掉重建，點擊憑空消失。修法：沒有拖曳動作時直接 return，
  不觸碰狀態、不重繪。

## v1.1.0 — 2026-06-30

**改善：行動裝置版面問題**

- `data-theme` 從 `.app` 移到 `<html>`，修正 CSS 變數（`--bg2` 等）無法往上傳給
  `body`/`#root` 造成的白底露出
- `100vh` 改用 `100dvh`（+ fallback），修正手機瀏覽器網址列伸縮造成的版面裁切/跳動
- 首次載入加一段內嵌 script 提前設定主題，避免白色閃一下才變深色（FOUC）
- 下拉同步／手動同步在還沒設定 token 時，改成溫和提示引導去設定頁，不再顯示嚇人的紅色
  失敗橫幅
- 設定頁「從/到」日期匯出範圍改成上下排列，修正並排時在窄螢幕手機上溢出卡片的問題

## v1.0.2 — 2026-06-30

**Bug 修正：無法用 `file://` 直接開啟，也無法在 Arc/Chrome 真實點擊互動**

- 拿掉 ES module（`type="module"`），全部改成普通 `<script src>`，`file://` 雙擊打開
  也能跑，不必透過本機伺服器
- 找到並修掉「你是誰呢」歡迎視窗打不開的根因：輸入框的 `onchange` 會觸發整頁重繪，重繪
  發生在使用者點下一步按鈕的手勢中間，導致點擊事件送不到已經被替換掉的按鈕節點上——這個
  bug 後來才發現其實跟 v1.1.1 是同一類問題，只是這次先在單一輸入框上修掉

## v1.0.1 — 2026-06-30

**改善：專案資料夾路徑排除疑慮**

- 專案資料夾名稱從 `baby log`（含空白）改為 `babylog`，排除路徑空白造成 `file://` 讀取
  問題的可能性（後來證實真正原因是 v1.0.2 記載的重繪問題，但保留這個改動）

## v1.0.0 — 2026-06-30

**初始版本**

- 首頁：快速記錄喝奶（母乳＋配方混合）／排便／尿尿，下一餐預測（最近 N 餐間隔中位數）
- 活動時間軸：今天／本週／本月／本年，長按拖曳調整時間，自動收合長空檔
- 統計：餵養摘要、照顧者分擔排行、WHO LMS 成長百分位曲線（體重/身高/頭圍，0–24 月，取自
  CDC 代管的官方 WHO 資料）
- 紀錄列表、編輯、刪除（軟刪除墓碑機制）
- 設定：寶寶資料、本機照顧者身分、日夜主題、事件時長、預設奶量、Alarm 微調
- CSV 匯出（Google 日曆匯入格式）
- GitHub Contents API 同步（pull-merge-push + SHA 衝突重試）+ 每日備份快照（v2.0.0 已換掉）
- 部署到 GitHub Pages：https://cs90s203.github.io/babylog/
